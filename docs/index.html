Honest note

This page is a demo of the mechanism, not a secured service. The key lives in your browser here, so anyone could read it; in production it stays on a server (or a per-tenant secret). What's genuinely offline is authenticity: a key-holder decodes and rechecks the tag locally, no database lookup. Two honest limits follow from the symmetric key. First, verifying implies forging: whoever can validate can also mint, so the key stays with the trusted merchant, and safe validation on untrusted terminals would need public-key signatures. Second, single-use needs state shared across validators: one offline device enforces it with a local ledger, but N offline terminals each catch only their own replays and would have to sync. The cipher is home-grown, so for real money the unforgeability rests on standard HMAC; the Enigma does the format and the flavour.

Built on the open EnigmaR engine, a standalone Enigma-style cipher. The mechanism is open; the only secret is the key.
